
## Что строим

Кликабельный мобильный web-прототип закрытого женского сообщества «Женское общество». Все данные — mock (TypeScript-объекты в коде), без бэкенда. Полностью оптимизировано под мобильный экран (390×844), но работает и в десктоп-превью.

## Подход к дизайну

Сначала сгенерирую **3 визуальных направления** в стилистике premium wellness / женственный минимализм (палитра: нежно-розовый, природный зелёный, молочный фон, мягкие градиенты). Все три используют одну палитру, но отличаются композицией, плотностью, типографикой и характером:

1. **Soft Sanctuary** — максимум воздуха, тонкая типографика, скругления, ощущение спа-приложения
2. **Editorial Bloom** — журнальная вёрстка, крупные заголовки serif, обложки-карточки с фото
3. **Cozy Circle** — тёплый, более плотный, иллюстративный, дружелюбные иконки, ощущение клуба

Вы выберете одно → по нему строится всё приложение.

## Объём первой итерации

**Полностью прорабатывается роль «Участница»** + переключатель ролей в шапке/меню профиля для будущих ролей (Наставник, Куратор, Администратор — заглушки-экраны с пометкой «в разработке»).

### Экраны Участницы (все кликабельные)

- **Онбординг** (3–4 шага): приветствие → анкета (имя, город, возраст, статус, деятельность, интересы) → выбор до 3 актуальных тем → готово
- **Главная** `/`: приветствие, «Рекомендовано для вас», ближайшие мероприятия, рекомендуемые наставники, группы сопровождения
- **Темы** `/topics`: 8 категорий с иконками-эмодзи → внутри темы список материалов (статьи/аудио/видео/практики) с фильтром и поиском → страница материала
- **Наставники** `/mentors`: каталог карточек → страница наставника (опыт, материалы, мероприятия, группы, отзывы, кнопки «Написать»/«Записаться»)
- **Мероприятия** `/events`: календарь/список с фильтрами (онлайн/офлайн/бесплатно/платно) → карточка мероприятия → «Записаться» / «Напомнить»
- **Сообщество** `/community`: табы «Знакомства», «Новости», «Общение» — современный чат-интерфейс
- **Группы сопровождения** `/groups`: список групп → карточка группы → «Подать заявку»
- **Профиль** `/profile`: фото, имя, город, интересы, сохранённое, мероприятия, группы, избранные наставники, **переключатель ролей** (тестовый)
- **Уведомления** `/notifications`: лента
- **Заглушки кабинетов** для Наставника / Куратора / Администратора — один экран на роль с превью разделов и пометкой «прототип»

### Навигация

Нижнее таб-меню (5 пунктов): 🏠 Главная · 📚 Темы · 👩 Наставники · 📅 Мероприятия · 👭 Сообщество. Профиль и уведомления — через иконки в шапке.

## Технические детали

- **Стек:** TanStack Start + Tailwind v4 + shadcn/ui (уже в проекте)
- **Маршруты:** файлы в `src/routes/` по схеме TanStack — отдельный route на каждый раздел
- **Данные:** `src/lib/mock-data.ts` с массивами наставников, мероприятий, групп, материалов, сообщений; локальный стейт через Zustand или React Context для роли/онбординга/сохранённого
- **Дизайн-токены:** палитра выбранного направления → `src/styles.css` (oklch, semantic tokens — без хардкода цветов в компонентах)
- **Изображения:** генерация фото для героев наставниц, обложек материалов, аватаров через imagegen (premium-стиль, мягкий свет)
- **Переключатель ролей:** селект в профиле, состояние в Context → меняет layout/нижнее меню
- **Шрифты:** подбираются под выбранное направление (вероятно sans + один акцентный serif для editorial-варианта)

## Что НЕ войдёт в первую итерацию

- Реальная авторизация и БД (выбран mock-режим)
- Глубокая проработка кабинетов Наставника/Куратора/Администратора — только превью-экраны с реальной структурой меню; полноценная реализация — следующими итерациями
- Реальная отправка сообщений / запись на мероприятия — все действия имитируются toast-уведомлениями
